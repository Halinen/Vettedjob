# Candidate profile (EXAMPLE — replace with your own)

This file is read by the OPTIONAL fit-scoring layer to judge how well a posting
matches you. The 4-layer legitimacy filter does NOT use it. If you only want
legitimacy filtering, set `fit_scoring.enabled = false` in config.json and this file
is never read.

## Summary
One paragraph: who you are, years of experience, the kind of role you want.

## Skills
- List your core skills, tools, and languages.

## Experience
- Role — Company — dates. One bullet on what you actually did and shipped.

## Education
- Degree, field, institution, year.

## Preferences
- Location, remote vs on-site, salary range, any deal-breakers.
